# A1-A5 检测项、规则与输出结果整理

> 更新时间：2026-04-17\
> 目的：根据 [a1-a5-source.md](./a1-a5-source.md) 整理 `A1`、`A2`、`A3`、`A4`、`A5` 的具体检测项、命中规则、是否需要配置以及输出结果示例，便于后续做产品定义、规则梳理与实现拆解。

## A1 检测项、规则与输出结果对照表

> 说明：`规则是否需配置` 为 `需配置` 的条目，表示需在后台规则配置中心由人工维护参数/策略（如阈值、适用范围、阻塞策略等）。

| 检测项 | 检测规则（命中条件） | 规则是否需配置 | 检测结果输出（示例） |
| -- | -- | -- | -- |
| 漏打卡 | payroll cutoff 前扫描打卡记录，员工存在应出勤记录但缺少 `clock-in` 或 `clock-out`。 | 否 | 输出员工、日期、班次、缺失字段（`clock-in` / `clock-out`）。建议动作：补卡或确认是否豁免。 |
| 加班超阈值（周 / 月） | 员工加班时长超过设定阈值，如 `>60h/周` 或 `>36h/月`。 | 需配置（周/月阈值、适用范围） | 输出本期加班时长、阈值、超出时长、风险级别。建议动作：复核 OT 记录并确认是否允许计薪。 |
| 连续通宵班 / 不合理排班模式 | 连续夜班天数、夜班组合或跨日班次模式超出策略阈值。 | 需配置（夜班定义、连续天数、例外人群） | 输出连续通宵班天数、触发规则、涉及班次。建议动作：复核排班或调整工时安排。 |
| 未审批加班影响发薪 | OT 记录状态未完成审批，但已进入本次 payroll calculation 范围。 | 否 | 输出员工、OT 小时、关联 payroll period、审批状态。建议动作：催审批或暂缓计薪。 |
| 提报加班与系统加班不一致 | 员工提报 OT 小时与系统计算 OT 小时差值超过允许偏差。 | 需配置（容差阈值、比对口径） | 输出提报值、系统值、差值、影响金额。建议动作：更正 OT 申报或确认计算口径。 |
| 打卡与请假重叠 | 同一员工同一时段同时存在有效打卡记录与有效请假记录。 | 需配置（重叠容差、允许重叠假别） | 输出重叠日期/时段、请假类型、打卡证据。建议动作：核对考勤与请假记录是否冲突。 |
| 法定节假日 / 休息日打卡 | 员工在 statutory holiday 或 rest day 存在打卡记录。 | 否 | 输出员工、日期、日期类型、打卡记录。建议动作：确认是否符合支付资格或触发合规提醒。 |

## A2 检测项、规则与输出结果对照表

> 说明：`规则是否需配置` 为 `需配置` 的条目，表示需在后台规则配置中心由人工维护参数/策略（如阈值、适用范围、阻塞策略等）。

| 检测项 | 检测规则（命中条件） | 规则是否需配置 | 检测结果输出（示例） |
| -- | -- | -- | -- |
| 待审批休假影响薪资 | 请假状态为 `Pending`，且请假日期落在本 payroll period，属于影响薪资计算的假别。 | 需配置（影响薪资假别、阻塞策略） | 输出员工、待审批天数、预计扣薪金额、审批人。建议动作：催审批并确认是否暂缓计薪。 |
| 请假负余额未自动扣薪 | 员工 leave balance < 0，且对应假别未启用 auto-deduction 或扣减配置缺失。 | 需配置（auto-deduct 开关、扣减公式、适用假别） | 输出当前余额、应扣金额、配置状态。建议动作：检查自动扣薪配置并补充处理。 |
| 假期折现资格异常 | 实际 leave encashment 结果与策略应得资格不一致，或超过折现上限。 | 需配置（资格条件、折现上限、适用假别） | 输出应得/实得、差额、命中策略。建议动作：核对政策配置并修正折现结果。 |
| Joiner / Leaver 按比例请假异常 | 入职 / 离职员工当期应得假期与系统按比例计算结果不一致。 | 需配置（计算公式、舍入规则、生效口径） | 输出服务天数、应得值、系统值、差值。建议动作：修正 proration 结果并复核起止日期。 |
| 异常请假模式 | 命中异常行为模式规则，如每周五固定请 `0.5` 天年假、病假当月 `>=X` 天。 | 需配置（模式模板、观察窗口、阈值） | 输出命中模式、观察窗口、实际次数 vs 阈值。建议动作：复核是否属于正常业务或例外情况。 |

## A3 检测项、规则与输出结果对照表

> 说明：`A3` 聚焦手工录入的 ad-hoc payment / deduction 项，包括 bonus、commission、reimbursement、manual deduction 等。

| 检测项 | 检测规则（命中条件） | 规则是否需配置 | 检测结果输出（示例） |
| -- | -- | -- | -- |
| 临时支付 / 扣款重复录入 | 同一员工、同一期间、相同项目、相同金额的 ad-hoc item 出现重复记录。 | 否 | 输出重复记录清单、重复次数、金额合计。建议动作：保留有效记录并删除或驳回重复项。 |
| 临时项金额异常值 | ad-hoc 金额显著高于该员工或该项目的历史平均值，超过统计阈值。 | 需配置（历史窗口、异常阈值） | 输出当前金额、历史均值、偏差幅度、风险级别。建议动作：复核是否为异常录入或特殊奖励。 |
| 政策上限违规 | 手工录入的津贴、补贴、报销或扣款超过公司政策上限，如 transport allowance cap。 | 需配置（项目上限、适用范围、周期） | 输出当前金额、政策上限、超出金额。建议动作：按政策修正或走特批流程。 |
| 大额临时支付缺少审批 | 高金额 ad-hoc payment 缺少规定的 approval workflow 或 sign-off。 | 需配置（大额阈值、审批层级） | 输出项目、金额、缺失审批节点。建议动作：触发补充审批流程后再入薪。 |
| 可变薪资未正确适用法定缴款 | bonus / commission 等 variable pay 未正确纳入 mandatory statutory contribution 计算。 | 需配置（法缴规则、适用薪资项） | 输出薪资项、法缴项目、预计少扣/多扣金额。建议动作：复核 statutory contribution 配置。 |

## A4 检测项、规则与输出结果对照表

> 说明：`A4` 偏向薪资预估和成本波动预警，核心是 payroll run 前的成本异常识别。

| 检测项 | 检测规则（命中条件） | 规则是否需配置 | 检测结果输出（示例） |
| -- | -- | -- | -- |
| 部门总成本环比异常 | 部门级 payroll total cost 相比上月变化超过设定阈值，例如 `>10% MoM`。 | 需配置（环比阈值、观察维度） | 输出部门、本月成本、上月成本、变化额、变化率。建议动作：确认是否属于预算内变化。 |
| 成本中心波动异常 | cost centre 维度的 payroll cost 在本期相较历史或上期出现显著波动。 | 需配置（成本中心口径、阈值、观察周期） | 输出成本中心、波动金额、变化率、受影响员工数。建议动作：下钻检查来源数据。 |
| 主要成本驱动项识别 | 本期成本变化主要由新入职、离职、奖金发放、调薪等事件驱动，且贡献度显著。 | 否 | 输出驱动因素清单及贡献金额，例如新员工、离职、bonus run、promotion adjustment。 |
| 奖金 / 调薪导致的异常增幅 | bonus run、年度调薪或组织调整导致部门成本增幅高于预期。 | 需配置（预期阈值、适用部门、事件类型） | 输出事件类型、关联部门、增幅金额、增幅比例。建议动作：确认是否已纳入预算或需特别说明。 |
| 按部门 / 员工类型成本分布异常 | 不同 employee type、department 或 site 的成本结构偏离历史分布或预期结构。 | 需配置（分组口径、结构偏差阈值） | 输出异常分组、当前占比、历史占比、差异说明。建议动作：进一步 drill-down 核对人员与薪资组成。 |

## A5 检测项、规则与输出结果对照表

> 说明：`A5` 为 post-payroll calculation anomaly detection，以下条目根据源文档中明确列出的 `9` 类异常类型整理。

| 检测项 | 检测规则（命中条件） | 规则是否需配置 | 检测结果输出（示例） |
| -- | -- | -- | -- |
| 净薪资与 6 个月均值差异异常 | 员工本期 net pay 相对近 `6` 个月滚动平均值的差异超过设定百分比阈值。 | 需配置（lookback period、差异阈值） | 输出当前净薪资、6 个月均值、差异金额、差异比例、风险级别。 |
| 同行群体薪资异常值 | 员工薪资相对同角色、同职级、同地点 peer group 偏离超过统计阈值。 | 需配置（peer group 定义、标准差阈值） | 输出员工、peer median / mean、偏离幅度、异常说明。 |
| 缺失或异常扣款 | 应有的经常性法定扣款缺失，或在无合理原因时出现异常扣款变化。 | 需配置（mandatory deduction mapping、豁免条件） | 输出缺失/异常的扣款项、上期金额、本期金额、可能原因。 |
| 意外津贴无权益记录 | 本期出现 allowance，但 HR entitlement 中无对应记录，或员工历史上从未出现过该津贴。 | 需配置（entitlement mapping、首现策略） | 输出津贴项、当前金额、 entitlement 状态、历史出现情况。 |
| 净薪资绝对下限 / 上限突破 | 员工 net pay 超出绝对最小值或最大值阈值，如 `<3500` 或 `>25000`。 | 需配置（floor / ceiling threshold） | 输出员工、净薪资、阈值、超出金额。建议动作：优先排查录入或公式错误。 |
| 税额差异异常 | 本期 tax withheld 相比近 `6` 个月平均值出现显著波动。 | 需配置（lookback period、差异阈值） | 输出本期税额、历史均值、差异比例、可能原因，如 tax code 错误或 AW 漏计。 |
| 加班金额 / 工时异常激增 | OT hours 或 OT-related pay 偏离同群体常态，超过设定标准差阈值。 | 需配置（peer group、标准差阈值） | 输出本期 OT、群体均值、偏离幅度、风险说明。 |
| 新增工资项未获审批 | 员工首次出现新的 pay component，且缺少 HR approval 或合法来源说明。 | 需配置（新工资项识别口径、审批规则） | 输出新增工资项、金额、首次出现期间、审批状态。 |
| 经常性扣款变动超阈值 | recurring deduction 金额相对前期变化超过设定阈值。 | 需配置（变动阈值、适用扣款项） | 输出扣款项、前期金额、本期金额、变动金额、变动率。 |
